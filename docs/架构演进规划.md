# Serial Log 架构演进规划

## 背景

当前版本已经完成基础串口日志、多窗口显示、命令发送、命令组、AT 导入和日志保存能力。后续功能会涉及窗口编排、浮动停靠、跨 PC 协作，这些改动会影响 ViewModel、布局状态持久化和串口会话归属模型，需要提前规划。

## 目标功能

### 1. 串口窗口拖动排序和跨页拖动

- 支持单独的串口窗口在当前页内拖动调整顺序。
- 支持把串口窗口拖到上一页/下一页，完成跨页移动。
- 拖动后更新工作区配置，重启软件后保持排序。

设计要点：

- 串口窗口顺序应由 `SerialWindows` 集合顺序决定，不要只在 UI 层做临时排序。
- 跨页拖动需要明确目标插入位置，例如拖到页边触发自动翻页，或拖到分页按钮上切页后继续放置。
- 连接中的串口窗口允许移动，但不能因此重建串口会话。

### 2. 命令窗口四向停靠

- 命令窗口默认停靠在底部。
- 可停靠到顶部：菜单栏下方、所有串口窗口上方，不改变串口窗口内部布局。
- 可停靠到左侧或右侧：需要主布局从固定上下结构改成可配置 Dock 布局。
- 停靠位置保存到工作区配置。

设计要点：

- 增加 `CommandPanelDock` 配置：`Bottom`、`Top`、`Left`、`Right`。
- 主界面布局建议抽象为 Shell Layout，串口区域和命令区域作为两个可重排区域。
- 左右停靠时，命令区需要适配窄宽度，避免现有横向控件被挤压。

### 3. 命令窗口浮动到桌面

- 命令窗口可从主窗口拖出，形成独立桌面窗口。
- 浮动后主窗口串口区域自动扩大。
- 浮动窗口可拖回主窗口，恢复停靠。

设计要点：

- 命令区域不能只写死在 `MainWindow.xaml` 中，应抽出独立 `CommandPanelView` 和对应 ViewModel。
- 主窗口和浮动窗口共享同一个命令发送 ViewModel，避免两套状态。
- 浮动窗口关闭时应定义行为：关闭即回到底部停靠，或只隐藏浮动窗口。
- 工作区需要保存命令窗口状态：停靠/浮动、浮动位置、尺寸。

### 4. 跨 PC 协作模式

- 默认本地模式：行为与当前版本一致。
- 联机模式：可以和多台 PC 协作。
- 联机模式下串口窗口需要显示 PC 标识。
- 不同 PC 的窗口最好用边框颜色区分。

设计要点：

- 需要引入 `WorkspaceMode`：`Local`、`Networked`。
- 串口窗口模型增加 `OwnerPcId`、`OwnerPcName`、`OwnerColor`。
- UI 上每个串口窗口的边框颜色由 PC 标识决定，本机窗口和远端窗口一眼可区分。
- 网络协议需要明确：
  - 发现机制：手动配置 IP，还是局域网广播发现。
  - 数据同步：只同步日志展示，还是允许远程发送命令。
  - 权限模型：哪些 PC 可以控制串口，哪些 PC 只能查看。
  - 断线行为：远端 PC 掉线后窗口如何保留和提示。

## 建议的架构拆分

### UI 层

- `MainWindow`：只负责 Shell 和窗口生命周期。
- `SerialWorkspaceView`：串口窗口网格和分页。
- `SerialWindowView`：单个串口窗口。
- `CommandPanelView`：命令发送、命令组、AT 导入。
- `FloatingCommandWindow`：浮动命令窗口。

### ViewModel 层

- `MainViewModel`：协调工作区、布局状态和全局命令。
- `WorkspaceLayoutViewModel`：管理串口窗口顺序、分页、命令面板停靠状态。
- `CommandPanelViewModel`：承载命令发送、命令组和 AT 导入逻辑。
- `SerialWindowViewModel`：保持串口会话状态，不因 UI 拖动或分页变化重建。
- `CollaborationViewModel`：后续管理 PC 节点、连接状态和远端窗口。

### Core 层

- 保持串口、日志、命令组、AT 解析等逻辑继续放在 Core。
- 新增协作能力时，网络协议和消息模型也应放在 Core，UI 层只订阅状态。

## 分阶段实施建议

1. 先抽出 `CommandPanelView` 和 `CommandPanelViewModel`，让命令区从 `MainWindow` 解耦。
2. 实现串口窗口拖动排序和排序持久化。
3. 实现命令区顶部/底部停靠。
4. 改造主布局，支持命令区左/右停靠。
5. 实现命令区浮动窗口。
6. 设计跨 PC 协作协议和本地模拟测试。
7. 实现联机模式、PC 标识和窗口边框颜色。

## 测试重点

- 拖动串口窗口后，连接中的串口不应断开。
- 跨页拖动后，分页、目标发送列表、命令组目标列表都要同步。
- 命令区停靠/浮动后，发送历史、命令组、AT 导入状态不丢失。
- 重启软件后恢复窗口排序、命令区位置和浮动窗口尺寸。
- 联机模式下，本机与远端串口窗口的命令权限、断线提示和颜色标识明确。
