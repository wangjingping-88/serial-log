# 串口 Log 工具使用说明

## 功能概览

Serial Log 是一个 Windows 串口日志桌面工具，基于 .NET 8 WPF。它适合多串口、多设备联调场景，重点能力包括：

- 多串口日志窗口：一页最多显示 3 x 2 个窗口，超过后自动分页。
- 顶部快速操作：新增页、删除页、连接全部、断开全部、日志目录、分页切换。
- 多机协作：主机汇总客户端串口窗口和日志状态，并可向客户端远端窗口下发命令。
- 命令区：支持单条命令、命令组、循环发送、历史记录、AT 命令导入和命令集管理。
- 命令区位置：支持停靠在下、上、左、右，也支持浮动成独立窗口。
- 日志保存：按连接会话自动保存，默认根目录为 `D:\serial-log-data\logs`。
- 版本识别：标题区显示应用版本和协作协议版本，便于多电脑排查版本不一致。

## 安装方式

推荐使用便携版：

1. 下载 GitHub Release 中的 `SerialLog-*-win-x64-portable.zip`。
2. 解压到任意目录，例如 `D:\tools\SerialLog`。
3. 双击 `SerialLog.App.exe` 启动。

如果使用 MSIX 安装包：

1. 同时下载 `SerialLog-*-win-x64.cer` 和 `SerialLog-*-win-x64.msix`。
2. 双击 `.cer`，选择“安装证书”。
3. 存储位置选择“当前用户”。
4. 选择“将所有的证书都放入下列存储”，证书存储选择“受信任的人员”。
5. 证书导入完成后，再双击 `.msix` 安装。

MSIX 必须有可信签名证书。当前 Release 会随包提供测试签名证书；如果不想导入证书，直接使用便携版 ZIP。

## 顶部工具栏

顶部从左到右主要包含：

- 应用标题与版本：显示 `Serial Log`、应用版本和协作协议版本。
- 页面操作：`新增页`、`删除页`。当前页为空时才能删除页。
- 串口操作：`连接全部` 会连接所有已配置端口且未连接的本地窗口；`断开全部` 会断开所有本地窗口。
- 多机协作：选择本地、主机、客户端模式，设置本机名称、颜色、主机 IP 和端口。
- 命令区位置：`下`、`上`、`左`、`右` 控制命令区停靠方向。
- 日志目录：设置自动保存日志的根目录。
- 分页：`上一页`、页码、`下一页`。串口窗口也可以拖到上一页/下一页按钮上跨页移动。

## 串口窗口

每个串口窗口标题栏包含：

- 发送目标复选框：勾选后参与单条命令发送。
- 窗口名称：可直接编辑，例如 `中心`、`R1`、`R2`。
- 端口选择：选择 COM 口；如果 Windows 暂时没有上报已保存端口，工具仍会保留该端口名。
- 波特率：默认 `115200`，可直接编辑。
- `刷新`：刷新当前窗口可选端口。
- `连接` / `断开`：连接或断开当前窗口串口。
- 自动保存复选框：控制当前窗口是否写入日志文件。
- `清空`：清空当前窗口显示日志，不删除已保存文件。
- `删除`：删除当前窗口，并同步命令区目标列表。
- `放大`：当前页有空槽时可纵向放大窗口，再次点击还原。
- 拖动：按住标题栏空白区域或“拖动”文字可调整窗口顺序。

窗口右上角状态点表示连接状态：

- 红色：未连接。
- 绿色：已连接。
- 黄色：端口占用、连接异常等警告状态。

远端窗口会显示来源电脑的颜色和名称。主机端看到的远端窗口不能直接连接本机串口，但可以接收客户端上报日志，并可作为命令发送目标。

## 日志保存

- 默认日志根目录：`D:\serial-log-data\logs`
- 每次连接都会创建新的会话目录，避免多次连接的日志混在一起。
- `连接全部` 会让本次批量连接的所有窗口共用一个会话目录。
- 单个窗口连接会为该窗口创建独立会话目录。
- 会话目录格式：`日志根目录\yyyy-MM-dd\yyyyMMdd_HHmmss_fff\`
- 日志文件名优先使用窗口名称。
- 单文件超过 50 MB 后自动滚动编号。
- Windows 中 `COM1` 到 `COM9` 是保留设备名，如果窗口标题正好是这类名称，日志文件名前缀会自动转换为安全名称。

## 命令区布局

命令区由两部分组成：

- 左侧公共区：`目标窗口`、`历史记录`、`命令导入`。
- 右侧命令区：`单条命令`、`命令组`。

当命令区停靠在左侧或右侧时，布局会自动改成上下结构，便于窄宽度显示。

命令区右上角的 `浮动` 可以把命令区弹出为独立窗口。浮动窗口关闭或点击恢复后，会回到原来的停靠位置。

## 单条命令

1. 在 `单条命令` 输入框输入命令，例如 `AT`。
2. 选择换行方式：`None`、`Cr`、`Lf`、`CrLf`，默认 `CrLf`。
3. 在 `目标窗口` 标签页勾选目标窗口。
4. 点击 `发送`。

发送逻辑：

- 只会向已勾选且已连接的目标发送。
- 未连接目标会跳过。
- 发送成功的命令会进入 `历史记录`，最多保留 100 条。
- 点击历史记录会自动回填：当前在单条命令页时填入单条命令输入框；当前在命令组页时填入命令组编辑框。

单条命令也支持循环发送：

- `循环间隔(ms)`：每轮发送之间的等待时间。
- `循环次数`：大于 0 时执行指定轮数；为 0 时一直循环，直到点击 `停止循环`。

## 命令组

命令组用于按顺序发送多条命令。

命令组设置包含：

- 命令组名。
- 发送对象：下拉选择当前命令组。
- 组内间隔：组内相邻两条命令之间的等待时间，默认 500 ms。
- 换行方式：默认 `CrLf`。
- 执行：只执行一轮命令组。
- 循环间隔：每轮命令组之间的等待时间。
- 循环次数：大于 0 时执行指定轮数；为 0 时一直循环，直到点击 `停止循环`。

目标选择：

- 切换到 `命令组` 标签后，左侧 `目标窗口` 会显示当前命令组自己的目标列表。
- 每个命令组有独立目标选择，不影响单条命令目标。

命令列表：

- 在命令输入框编辑命令后点击 `加入命令`。
- 支持拖动调整命令顺序。
- 支持右键删除或按 Delete 删除。
- 从历史记录或命令导入中点击 `入组` 时，只会先填入命令编辑框，方便修改参数后再加入命令组。

## AT 命令导入

`命令导入` 标签页支持多套命令集：

- 左侧下拉选择命令集。
- 右侧文本框可重命名当前命令集。
- `新建` 创建新的命令集。
- `删除` 删除当前命令集；至少保留一套。

导入入口：

- `导入AT`：选择文件并覆盖当前命令集。
- `追加AT`：选择文件并追加到当前命令集，完全相同命令会去重。
- `自定义`：手动输入命令，一行一条。
- `填单条`：把选中的 AT 命令填入单条命令输入框。
- `入组`：把选中的 AT 命令填入命令组编辑框。

支持的文件内容：

```text
# 注释行会被忽略
AT
AT+GMR
AT+RESET
```

也支持字面量 `\r\n` 分隔的命令列表：

```text
AT\r\nAT+GMR\r\nAT+RESET\r\n
```

也支持从 `.c/.h` 文件中的 `AT_CMD_EXPORT(...)` 提取命令：

```c
AT_CMD_EXPORT("AT+ROLE", "=<role[0-2]>", mesh_at_role_test, mesh_at_role_query, mesh_at_role_setup, NULL);
AT_CMD_EXPORT("AT+NB", NULL, NULL, mesh_at_neighbor_query, NULL, NULL);
```

导入结果：

```text
AT+ROLE=<role[0-2]>
AT+NB
```

## 多机协作

当前多机协作是“主机汇总”模型，不是所有电脑完全镜像同一工作区：

- 每台电脑只管理自己的本地串口窗口。
- 客户端把本机窗口快照和日志行上报给主机。
- 主机显示本机窗口 + 客户端远端窗口。
- 主机可以向客户端远端窗口下发命令。
- 客户端会发送心跳；断线后进入等待重连并自动重试。

快速验证：

1. 主机电脑：顶部 `多机协作` 选择 `主机`。
2. 确认主机 IP 和端口，点击 `启动主机`。
3. 客户端电脑：顶部 `多机协作` 选择 `客户端`。
4. 填写主机 IP 和同一端口，点击 `连接主机`。
5. 连接后，主机端会看到客户端远端窗口和来源颜色。
6. 在主机端勾选远端窗口并发送命令，可下发到客户端对应串口。

注意事项：

- 多台电脑需要在同一局域网内互通。
- Windows 防火墙需要允许主机端监听端口。
- 如果多机协作异常，先确认各电脑标题区显示的协议版本一致。

## 工作区配置

工作区配置默认保存到：

```text
D:\serial-log-data\workspace.json
```

保存内容包括：

- 日志根目录。
- 串口窗口标题、端口、波特率、分页、自动保存状态。
- 多机协作模式、本机名称、本机颜色、主机 IP、端口。
- 命令区停靠位置和浮动状态。
- 发送历史。
- AT 命令集。
- 命令组配置。
