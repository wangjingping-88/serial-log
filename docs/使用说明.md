# 串口 Log 工具使用说明

## 功能概览

- Windows 桌面应用，基于 .NET 8 WPF。
- 主窗口默认最大化打开，整体使用浅色背景。
- 中间区域按 3 列 x 2 行显示串口日志窗口，超过 6 个窗口后使用分页切换。
- 每条完整日志行在 PC 接收时添加毫秒级时间戳，格式为 `[yyyy-MM-dd HH:mm:ss.fff]`。
- 底部支持单条命令多目标发送，以及命令组分组发送。
- 支持 AT 文件导入，一行一条命令，空行和 `#` 注释行会被忽略；也支持从 `.c/.h` 文件里的 `AT_CMD_EXPORT` 自动提取命令。
- 支持从串口日志中导入 AT 命令，例如先发送 `AT&L` 列出命令，再点击“从日志导入”。
- 支持手动导出当前窗口日志，并支持按连接会话自动保存日志。

## 日志窗口

每个日志窗口包含：

- 发送目标复选框：勾选后会参与底部“单条发送”。
- 标题栏：常驻显示窗口标题、连接状态和接收行数。
- 快连：标题栏右侧可直接连接当前窗口配置的串口，不需要展开配置区。
- 清空：标题栏右侧可直接清空当前窗口显示日志，不删除已保存文件。
- 新增/删除：顶部仍可新增窗口；当前页不足 6 个窗口时，空白槽位中央会显示 `+`，点击即可新增窗口。标题栏右侧可删除当前窗口；删除后分页和底部发送目标会自动同步。
- 配置区：默认折叠，点击“配置”展开后可设置串口号、波特率、连接、自动保存、查找、清空和导出。
- 日志区：默认白底黑字，便于长时间阅读串口文本。
- 查找：输入关键词后，匹配行会高亮。
- 清空：清空当前窗口内存中的显示日志，不删除已保存文件。
- 导出：把当前窗口显示的日志导出到手动选择的文件。

## 命令发送

### 单条发送

1. 在底部“单条发送”输入命令。
2. 选择换行方式：`None`、`Cr`、`Lf`、`CrLf`，默认 `CrLf`。
3. 在“目标窗口”勾选目标串口。
4. 点击“发送”。

发送成功的命令会进入发送历史。

## 快速连接

- 顶部 `连接全部`：会尝试连接所有已经配置了端口号、且当前未连接的窗口。
- 顶部 `断开全部`：会断开所有串口窗口，并停止这些窗口的自动重连。
- 每个窗口标题栏的 `快连`：只连接当前窗口。
- 重新打开软件后，如果工作区里保存了端口配置，可直接使用这两个入口快速恢复连接。

### 命令组发送

命令组包含：

- 名称
- 目标串口列表
- 命令列表
- 命令间隔，默认 500 ms
- 换行方式，默认 `CrLf`

执行命令组时，工具会按命令列表顺序发送；每条命令会发送到该组勾选的所有已连接目标。未连接目标会跳过，并在底部状态栏显示统计。

## AT 文件格式

示例：

```text
# 注释行会被忽略
AT
AT+GMR
AT+RESET
```

导入后可选中命令并加入当前命令组。

导入按钮行为：

- `导入 AT 文件`：覆盖右侧当前 AT 命令列表。
- `追加 AT 文件`：把新文件中的命令追加到当前列表末尾，已存在的完全相同命令会自动去重，不会重复添加。
- 追加时如果遇到同名但参数不同的命令，例如 `AT+ROLE=<role>` 和 `AT+ROLE=<role[0-2]>`，会弹窗询问保留当前命令还是使用新文件命令。

也可以直接导入 C 源码中的 AT 导出宏：

```c
AT_CMD_EXPORT("AT+ROLE", "=<role[0-2]>", mesh_at_role_test, mesh_at_role_query, mesh_at_role_setup, NULL);
AT_CMD_EXPORT("AT+NB", NULL, NULL, mesh_at_neighbor_query, NULL, NULL);
```

导入结果会显示为：

```text
AT+ROLE=<role[0-2]>
AT+NB
```

如果设备支持 `AT&L` 列出所有命令，可以先把该命令发到串口，等日志窗口显示命令列表后，在右侧点击“从日志导入”。工具会从当前勾选的窗口日志中提取 `AT...` 命令；如果没有勾选窗口，则从全部窗口日志中提取。

## 保存规则

- 默认日志根目录：`D:\serial-log-data\logs`
- 每次连接都会创建新的会话目录，避免多次连接的日志混在一起。
- `连接全部` 会让本次批量连接的所有串口共用一个会话目录；单个窗口 `快连` 会为该窗口单独创建一个会话目录。
- 会话目录格式：`日志根目录\yyyy-MM-dd\yyyyMMdd_HHmmss_fff\`
- 日志文件名优先使用你在窗口标题中重命名的串口名，例如 `主控串口_20260630_001.log`。
- 单文件超过 50 MB 后自动滚动编号。
- Windows 中 `COM1` 到 `COM9` 是保留设备名，如果窗口标题正好是这类名称，日志文件名前缀会自动转换为类似 `COM3_` 的安全名称。

## 工作区配置

工作区配置默认保存到：

```text
D:\serial-log-data\workspace.json
```

保存内容包括：

- 日志根目录
- 串口窗口标题、端口、波特率、自动保存状态
- 当前分页
- 发送历史
- 命令组配置
